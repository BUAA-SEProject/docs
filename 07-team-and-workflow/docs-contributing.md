# [占位] 文档协作规范

> [占位] 当前文档尚未进入正式编写阶段。本页仅保留结构化草案，不可直接作为课程提交稿、开发依据或答辩材料使用。

## 基本原则

- 文档和代码一起演进
- 决策写入文档，不只存在口头讨论中
- 一页只解决一个主题，命名清晰可检索

## 编写建议

- 标题直接说明主题
- 先写结论，再写细节
- 对版本、范围、前提条件写清楚
- 重要概念统一用词

## GitBook 维护注意事项

- 新文件加入后同步更新 `SUMMARY.md`
- 保持层级不要过深，优先控制在两层
- 页面之间尽量建立交叉链接，减少重复说明

## 提交流程建议

1. 修改 Markdown 内容
2. 检查导航与相对链接
3. 提交代码评审
4. 同步到 GitBook 并确认展示效果
