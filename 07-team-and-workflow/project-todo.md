# 项目 Todo

本页根据现有项目 Todo 整理，适合作为 GitBook 中的项目推进总览。

## 1. 项目启动

- 明确项目名称、定位和目标用户
- 明确技术栈、主端形态和部署方式
- 确认核心需求与第一版范围
- 建立仓库、分支规范、任务管理工具和文档目录
- 完成团队分工与风险清单

## 2. 过程文档

- 完成软件开发计划书
- 完成软件需求规格说明书
- 完成概要设计说明书
- 完成详细设计说明书
- 完成测试报告、部署文档、用户手册

## 3. 研发工作

- 完成页面信息架构
- 完成 API 设计与数据库设计
- 完成前端、后端、评测模块主链路
- 完成联调、测试和部署验证

## 4. 验收准备

- 核对核心需求覆盖情况
- 准备演示脚本、视频和答辩材料
- 检查系统稳定性与文档一致性

## 5. 加分项

- 沉淀可复用 Skills
- 增加 AI 助教或智能答疑
- 完善 UI/UX 与可观测性
