# 错误码规范

## 设计原则

- 业务错误与系统错误分层
- 错误信息要便于前端展示和日志检索
- 同类错误保持统一命名

## 建议格式

| 错误码 | 含义 | 说明 |
| --- | --- | --- |
| 0 | 成功 | 请求处理成功 |
| 40001 | 参数错误 | 请求参数校验失败 |
| 40101 | 未登录 | 缺少或无效凭证 |
| 40301 | 无权限 | 角色无访问权限 |
| 40401 | 资源不存在 | 目标对象不存在 |
| 40901 | 状态冲突 | 当前对象状态不允许该操作 |
| 50001 | 系统异常 | 未知服务端错误 |
| 50011 | 评测失败 | 评测任务执行异常 |

## 建议补充

- 模块前缀规则
- 前端提示语规范
- 可重试与不可重试错误区分
