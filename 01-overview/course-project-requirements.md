# 课程大作业要求

本页整理自课程发布的大作业要求，作为项目文档建设的基线。

## 过程文档交付

### 第七周周日前

- 软件开发计划书

### 第八周周日前

- 软件需求规格说明书

### 第十周周日前

- 软件概要设计说明书

### 第十二周周日前

- 软件详细设计说明书

### 第十五周周日前

- 测试报告
- 部署文档
- 用户手册
- 源代码和可执行代码

## 项目交付重点

- 核心需求覆盖度
- 系统可用性
- 答辩展示清晰度
- 团队分工合理性

## 本项目建议重点

结合“在线教学与实训平台”题目，建议优先保证以下主链路完整：

1. 用户登录与角色鉴权
2. 教师建课与发布作业/实验
3. 学生参与课程并在线提交
4. 自动评测与结果回传
5. 教师批改与成绩统计
6. 通知与结果可视化

## 文档建设约束

- 过程文档必须连续，并与最终系统实现相对应。
- 需求、设计、测试和部署文档应能证明核心需求是如何逐步落地的。
- 若使用大模型辅助开发，需要单独维护使用说明、Session 和相关 Skill 材料。
- 团队成员应能解释自己负责的文档与代码内容，避免“生成但不理解”。
