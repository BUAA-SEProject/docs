## 2. 项目背景与目标

本项目面向高校教学场景，建设一个支持课程管理、实验实训、自动评测和成绩统计的在线教学与实训平台。

阶段目标：

- 明确项目边界与核心需求
- 建立可持续协作的开发与文档流程
- 保证课程要求的过程文档按期交付

## 3. 范围与边界

### 本次迭代必须完成

- 教师与学生双角色主流程
- 课程、作业/实验、提交、评测、批改、成绩主链路
- 过程文档、部署文档、用户手册齐全

### 可选增强项

- AI 助教 / 智能答疑
- 管理员仪表盘
- 多端适配

## 7. 风险管理

| 风险 | 表现 | 应对方案 |
| --- | --- | --- |
| 范围膨胀 | 功能过多导致延期 | 锁定 P0 核心需求 |
| 架构过重 | 早期设计过度复杂 | 先保证主链路可实现 |
| 文档滞后 | 设计与实现脱节 | 每周同步更新文档 |
| 环境不稳 | 演示时部署失败 | 提前完成一键部署演练 |

## 8. 质量保证

- PR 评审与关键设计评审
- 单元测试、接口测试、主流程冒烟测试
- 文档一致性检查
- 演示前全链路回归

## 9. 配置与沟通管理

- 分支建议：`main` / `dev` / feature 分支
- 提交信息建议：模块名 + 动作 + 结果
- 每周例会一次，必要时增加专题评审
- 所有正式决策写入本 GitBook 文档站
