# 过程文档

本部分存放课程要求的正式交付件。

| 文档 | 状态 | 说明 |
| --- | --- | --- |
| 软件开发计划书 | 已完成 | 明确范围、边界、阶段出口与验收口径 |
| 软件需求规格说明书 | 已完成 | 形成正式需求、非功能需求、数据需求与验收矩阵 |
| 软件概要设计说明书 | 已完成 | 定义系统上下文、架构视图、模块划分和部署方案 |
| 软件详细设计说明书 | 已完成 | 定义页面、服务、状态机、数据流与异常处理 |

| 部署文档 | 已完成 | 形成环境、部署、初始化、回滚和排障步骤 |
| 用户手册 | 已完成 | 形成管理员、教师、学员三类操作说明 |
| 大模型使用说明 | 已完成基线版 | 形成使用边界、复核规则、记录模板和答辩材料要求 |

## 使用规则

- 阅读顺序建议为：计划书 -> SRS -> 概要设计 -> 详细设计 -> API / 数据库 / 测试 / 部署。
- 需求边界变化应先修改 SRS，再同步设计和测试文档。
- 设计实现变化但不改变需求边界时，应优先更新概要设计、详细设计、研发文档和 API 文档。
- 测试报告和大模型使用说明中的“执行证据”部分应随着开发推进持续补充，而不是在答辩前一次性补写。
