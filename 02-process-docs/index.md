# 过程文档

本部分存放课程要求的正式交付件。每篇文档都应能够独立阅读：正文直接说明本文件负责的范围、结论和关键依据，不把理解成本转移到其他文档。

## 当前状态

| 文档 | 状态 | 说明 |
| --- | --- | --- |
| 软件开发计划书 | 已完成 | 明确范围、边界、阶段出口与验收口径 |
| 软件需求规格说明书 | 已完成 | 形成正式需求、非功能需求、数据需求与验收矩阵 |
| 软件概要设计说明书 | 已完成 | 定义系统上下文、架构视图、模块划分和部署方案 |
| 软件详细设计说明书 | 已完成 | 定义页面、服务、状态机、数据流与异常处理 |
| 测试报告 | 已完成 | 形成测试范围、执行结果、压力测试结论、缺陷和残留风险 |

## 支撑附录

| 文档 | 状态 | 说明 |
| --- | --- | --- |
| 部署文档 | 已完成 | 形成环境、部署、初始化、回滚和排障步骤 |
| 用户手册 | 已完成 | 形成管理员、教师、学员三类操作说明 |
| 大模型使用说明 | 已整理 | 形成使用边界、复核规则、记录模板和答辩材料要求 |

## 使用规则

- 阅读顺序建议为：计划书 -> SRS -> 概要设计 -> 详细设计 -> 测试报告 -> 部署文档 -> 用户手册。
- 需求边界变化应先修改 SRS，再同步设计和测试文档。
- 设计实现变化但不改变需求边界时，应优先更新概要设计、详细设计和相关支撑附录。
- 测试报告必须直接写明功能、压力、部署验证的结论；原始日志和截图可归档保存，但不能替代报告正文。
