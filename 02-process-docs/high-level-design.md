# 软件概要设计说明书

- 版本：v0.1

- 负责人：[待补充]

## 2. 设计目标

通过模块化架构支撑课程管理、作业/实验、评测、批改和成绩统计等核心能力，并保证部署和演示的稳定性。

## 3. 总体架构

建议采用以下分层：

- 表现层：Web 前端、可选移动端
- 接口层：认证、课程、作业、评测、成绩、通知 API
- 业务层：课程服务、作业服务、评测服务、统计服务
- 数据层：关系数据库、对象存储、日志存储
- 平台层：部署、监控、CI/CD

## 4. 模块划分

| 模块 | 主要职责 |
| --- | --- |
| 用户与权限 | 登录、角色、权限控制 |
| 课程管理 | 课程、成员、资源、公告 |
| 作业与实验 | 发布、提交规则、状态流转 |
| 在线实验 | 运行环境、过程记录 |
| 自动评测 | 编译、执行、评分、日志 |
| 批改与成绩 | 人工批注、统计、导出 |
| 通知中心 | 系统通知与提醒 |

## 5. 关键设计决策

- 先保证单体应用内的模块清晰，再决定是否拆分服务
- API 统一版本与错误码规范
- 核心对象需要状态流转设计
- 演示环境优先保证可一键启动
