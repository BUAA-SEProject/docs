# 大模型使用说明

## 1. 文档信息

- 文档名称：AUBB（Academic Unified Builder Bench）大模型使用说明
- 版本：v1.0
- 状态：使用基线稿
- 更新日期：2026-04-14
- 适用范围：文档编写、方案分析、代码辅助、测试辅助、答辩材料准备

## 2. 使用目的与边界

- 大模型只能辅助分析、生成初稿、给出备选方案，不能替代人工验收。
- 所有进入仓库的文本、接口、设计和代码必须经过项目成员人工复核。
- 对课程评分有直接影响的内容，如核心需求、接口约束、测试结论，不得未经核查直接采用。

## 4. Session 与 Skill 材料

建议在仓库根目录下补充如下目录：

```text
llm-artifacts/
  sessions/
  prompts/
  skills/
  reviews/
```

归档要求：

- `sessions/`：保存关键会话导出或链接说明。
- `prompts/`：保存对正式输出有显著影响的提示词。
- `skills/`：保存团队自定义 Skill 文件。
- `reviews/`：保存人工复核记录和修订说明。

## 5. 人工复核说明

1. 明确本次使用的输入问题和预期输出。
2. 保存会话链接、会话 ID 或导出文本。
3. 由直接负责人检查内容是否与课程要求、SRS 和已有设计一致。
4. 若内容进入正式文档或代码，再由第二位成员进行交叉复核。
5. 在本文件中登记用途、模型、输出和复核人。

## 6. 风险控制

| 风险 | 控制措施 |
| --- | --- |
| 内容失真 | 以课程要求、SRS 和设计文档为唯一事实基线 |
| 文档堆砌 | 只保留能直接指导开发或答辩的内容 |
| 版权风险 | 不直接复制第三方受版权保护的大段内容 |
| 安全风险 | 不向外部模型输入密钥、账号、敏感个人数据 |
