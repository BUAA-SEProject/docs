# 测试报告

## 1. 文档信息

- 文档名称：在线教学与实训平台测试报告
- 版本：v1.0
- 状态：验证基线稿，待结合实现结果补实测证据
- 更新日期：2026-04-13
- 编写依据：SRS、概要设计、详细设计、测试策略
- 参考标准：ISO/IEC/IEEE 29119 系列

## 2. 测试目标

本报告用于验证以下内容：

1. P0 主链路是否完整可运行。
2. MUST 级需求是否存在阻断性交付缺陷。
3. 判题、成绩、权限和审计等高风险功能是否具备基本正确性。
4. 部署文档与用户手册是否能支撑演示与验收。

## 3. 测试范围

| 范围 | 内容 |
| --- | --- |
| 功能测试 | 登录、平台治理、课程、任务、提交、判题、批改、成绩、通知 |
| 接口测试 | 核心 REST API、权限控制、错误码一致性 |
| 集成测试 | 提交到评测、批改到成绩发布、导入到审计留痕 |
| 冒烟测试 | 演示账号、演示数据、关键页面和关键链路 |
| 非功能验证 | 基础响应时间、审计日志、关键指标、异常提示 |

## 4. 测试环境基线

| 项目 | 基线 |
| --- | --- |
| 前端 | `platform-web`，生产构建版 |
| 后端 | `platform-api`，开发或预发配置 |
| 判题 | `judge-worker` + `go-judge` |
| 数据库 | PostgreSQL 16 |
| 缓存 / 队列 | Redis 7 |
| 对象存储 | MinIO |
| 浏览器 | Chrome、Edge、Firefox 最新两个主要版本 |

## 5. 测试方法与策略

- 先执行接口级与页面级冒烟，再执行主链路集成测试。
- 对高风险能力采用“正常路径 + 异常路径 + 权限路径”三类用例。
- 所有执行证据统一保存在测试证据目录，便于答辩抽查。

## 8. 测试结论与残留风险

准入答辩和验收前，至少满足：

1. 所有 P0 缺陷关闭。
2. 主链路用例全部通过。
3. 关键证据已归档，包括截图、日志、接口响应和导出文件。
4. 部署文档经全新环境复验成功。
