# 研发文档

本部分从实现视角回答“如何编码、如何组织仓库、如何维护系统”。与过程文档相比，这里的内容更贴近工程落地。

## 文档分工

| 文档 | 关注点 |
| --- | --- |
| 总体架构 | 技术栈、分层、部署拓扑、跨模块约束 |
| 前端文档 | 路由、页面、状态管理、组件组织、交互约定 |
| 后端文档 | 模块结构、服务职责、权限、中间件、异步流程 |
| 判题与沙箱设计 | 评测链路、任务模型、资源限制、故障恢复 |
| 数据库设计 | 表结构、索引、关系与保留策略 |
| 工程规范 | 分支、提交、代码风格、测试、评审和交付标准 |

## 使用顺序

1. 先看 [总体架构](architecture-overview.md) 确认技术路线。
2. 再并行阅读 [前端文档](frontend.md)、[后端文档](backend.md) 和 [数据库设计](database.md)。
3. 编程任务相关功能必须额外阅读 [判题与沙箱设计](judge-and-sandbox.md)。
4. 开始编码前统一按 [工程规范](engineering-standards.md) 建立目录和提交规则。
