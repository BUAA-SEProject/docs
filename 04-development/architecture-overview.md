# [占位] 总体架构

> [占位] 当前文档尚未进入正式编写阶段。本页仅保留结构化草案，不可直接作为课程提交稿、开发依据或答辩材料使用。

## 架构目标

- 支撑教师与学生双角色主流程
- 保证作业/实验与评测链路清晰
- 保持实现简单、可演示、可维护

## 推荐逻辑分层

### 1. 客户端层

- Web 前端
- 可选移动端或小程序端

### 2. 应用层

- 认证与用户模块
- 课程模块
- 作业/实验模块
- 提交与评测模块
- 批改与成绩模块
- 通知模块

### 3. 基础设施层

- 关系数据库
- 文件存储
- 日志与监控
- 部署与容器运行环境

## 关键设计原则

- 先模块化单体，再视需要拆分服务
- 接口和错误码统一
- 状态流转显式建模
- 所有核心对象都能追踪操作记录
